Diamond problem (C++)

17 May 2020

Opinion:
Interestingly diamond problem is a most commonly asked interview question. But, In my experience developers won't encouter this scenario very often.

Diamond Problem:
Multiple inhertance is one of the useful feature in OOP programming. In multiple inheritance a derived class can be derived from multiple derived classes which already derived from the common base class. Lets take the below example, Player is a base class and Cricketer and Shooter are the derived classes. These two derived class will have the own copy of base class attributes.

SportsIcon is a new derived class which inherits attributes from both Cricketer and Shooter class which alreay inherited from Player class. Lets say if we are trying to create an object of SportsIcon class. The SportsIcon object couldn't decide which base class needs to be called. So it creates the copy of both class(see output in bottom).

How to fix:
To overcome the diamond problem, change base class inheritance to virtual inheritance(refer right image makred in white).
